# Tech Sanity Check

## What This Achieves

Phillip regarde le code et les choix techniques avec son œil de PDG qui connaît l'IT. Il cherche les absurdités, l'over-engineering, les dépendances inutiles, les choix qui vont coûter cher à maintenir. Quand il trouve des conneries, il le dit — avec toute la subtilité qu'on lui connaît.

## What Success Looks Like

- Identification des choix techniques qui impactent la rentabilité
- Dénonciation des absurdités avec des exemples concrets
- Propositions pragmatiques (pas de refactoring académique)
- Focus sur ce qui coûte du temps/argent, pas sur le style de code
